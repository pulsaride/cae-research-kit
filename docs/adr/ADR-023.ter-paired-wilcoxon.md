# ADR-023.ter — Disambiguation of the Wilcoxon test for H6-β

**Status:** PROPOSED — frozen at SHA-256 record time
**Date:** 2026-05-09
**Supersedes:** ambiguity in ADR-022 §3 / ADR-023 §4.6 only.
**Confidentiality:** PRIVATE — `.forge_private/` only.

---

## 1. Why this exists

ADR-022 §3 states the H₆ decision rule as:

> *Cohen's $d \geq 0.5$ AND Wilcoxon $p < 0.01$ over $n \geq 30$ disjoint
> seeds.*

It also frames $H_0$ as:

> *the bottleneck distance between persistence diagrams of (E₀ post-Ω) and
> (E₀ post-MarkovFit(Ω)) is statistically indistinguishable from the bootstrap
> distribution under the Markov fit.*

These two statements are not equivalent. The decision rule names a Wilcoxon
test but does not specify **which Wilcoxon test** (one-sample, paired,
two-sample), nor what null vector is paired against the real-vs-Markov
$\delta$ vector.

This ambiguity must be resolved **before** the adjudicator is written. ADR-022
itself is frozen and cannot be amended. ADR-023 inherited the ambiguity
verbatim. ADR-023.ter selects the operational test under the freeze clauses
of both prior ADRs, without altering their substantive scope.

## 2. Scope

Only the Wilcoxon test of ADR-022 §3 / ADR-023 §4.6 is bound here. Cohen's
$d$ remains as defined (effect size on the same paired sample), threshold
$0.5$ unchanged, $p$-threshold $0.01$ unchanged, $n=30$ unchanged, seed range
`[1200, 1229]` unchanged.

ADR-022 §8 anti-pattern *"Tweaking the metric mid-experiment"* applies; this
correction freezes the test BEFORE any code that reads its result is written.

## 3. The test — paired Wilcoxon on (real-vs-null) − (null-vs-null)

For each seed $s \in [1200, 1229]$:

1. **Real run.** Drive Agent A on $E_0(s)$. Record pressure trajectory
   $\mathbf{X}^{(s)}_\Omega$ and action sequence $\{a^{(s)}_t\}$ per
   ADR-023.bis §3.
2. **Markov fit.** $\hat{M}^{(s)} = $ `fit_markov`$(\{a^{(s)}_t\}, D, \alpha=1)$.
3. **Two independent null runs at the same seed.** Draw two action sequences
   $\{a^{0,A}_t\}, \{a^{0,B}_t\}$ from $\hat{M}^{(s)}$ using two distinct
   sampler streams seeded from $s$ (sampler stream 1 = $\mathrm{rng}(s)$,
   sampler stream 2 = $\mathrm{rng}(s + 10^6)$). Replay each on a fresh
   $E_0(s)$. Z-score both with the real run's $(\mu, \sigma)$ per ADR-023.bis
   §3.4 + §3.5.
4. **Distances.**
   - $\delta^{(s)}_{\Omega, M} = \mathrm{phase\_delta}(\tilde{\mathbf{X}}^{(s)}_\Omega, \tilde{\mathbf{X}}^{(s)}_{0,A})$
   - $\delta^{(s)}_{M, M} = \mathrm{phase\_delta}(\tilde{\mathbf{X}}^{(s)}_{0,A}, \tilde{\mathbf{X}}^{(s)}_{0,B})$

The vector of paired differences across seeds is
$$\Delta^{(s)} = \delta^{(s)}_{\Omega, M} - \delta^{(s)}_{M, M}$$

**Test:** `scipy.stats.wilcoxon(`$\Delta^{(s)}$`, alternative="greater")` — a
one-sided **paired Wilcoxon signed-rank test** that the real-vs-Markov
distance exceeds the Markov-self dispersion.

**Effect size:** Cohen's $d$ on the paired vector $\Delta^{(s)}$:
$$d = \frac{\overline{\Delta}}{\mathrm{stdev}(\Delta)}$$

**Decision rule (unchanged from ADR-022 §3, now operationalized):**
- $d \geq 0.5$ AND $p < 0.01$ ⇒ `H6_SIGNATURE_DETECTED`
- $d < 0.2$ ⇒ `H6_REJECTED`
- otherwise ⇒ `H6_INCONCLUSIVE`

## 4. Why this test (rationale, informational)

- The metric $\delta$ has irreducible noise floor from PD numerics
  (cf. ADT-H6-01 / ADT-H6β-01: even self-replay yields $\delta < 0.05$, not
  $0$). A one-sample Wilcoxon vs $0$ would conflate this noise with real
  signal and inflate type-I error.
- The Markov-self distribution $\delta^{(s)}_{M, M}$ is the natural null:
  it captures exactly the dispersion expected under $H_0$ (the Markov fit is
  the data-generating process).
- One-sided (`alternative="greater"`) because the doctrine asks whether real
  trajectories are *more* topologically distant from the Markov fit than the
  Markov fit is from itself. A "less" outcome would falsify in a different
  way and is captured by `d < 0.2 → REJECTED`.
- Pairing per seed eliminates seed-to-seed variance, which is the dominant
  noise source given `E0Config.grid_size=64, horizon=512` (cf. H₅ verdict
  variance structure).

## 5. Cost impact

H6-β.4 was costed at 2 E₀ runs per seed (1 real + 1 null) × 30 ≈ 60 runs.
ADR-023.ter requires 3 E₀ runs per seed (1 real + 2 nulls) × 30 ≈ 90 runs.
Single-seed timing (verified 2026-05-09): ~2 s for the full pipeline at
`grid_size=64, horizon=512, maxdim=1`, dominated by PD computation. Expected
total: 5–8 minutes.

## 6. Anti-patterns (in addition to ADR-022 §8 + ADR-023 §7)

- Switching to a one-sample test "because pairing introduces noise". The
  paired test is the bound test; deviation requires ADR-023.quater.
- Switching to a two-sided alternative. ADR-022 §3 is one-sided in spirit
  (the failure mode `d < 0.2 → REJECTED` already covers the other tail).
- Drawing additional null samples beyond `A` and `B` ("bootstrap with
  $k=1000$") in the main run. The frozen baseline is a single Markov-self
  pair, not a full bootstrap. Bootstrap analysis is allowed only as a
  POST-hoc diagnostic, plotted alongside the verdict, never as an
  alternative test.
- Changing sampler stream offsets ($s + 10^6$) without an ADR.
