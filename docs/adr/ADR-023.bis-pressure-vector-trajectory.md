# ADR-023.bis — Correction of ADR-023 §4.1 (phase space = pressure vector trajectory)

**Status:** PROPOSED — frozen at SHA-256 record time
**Date:** 2026-05-09
**Supersedes:** ADR-023 §4.1 only. All other ADR-023 sections remain in force.
**Confidentiality:** PRIVATE — `.forge_private/` only.

---

## 1. Why this exists

ADR-023 §4.1 referenced "scalars $U_t, F_t, R_t, P_t$ as already exposed by the
H₅ `RunResult` API". Reading the actual source confirmed this is factually
wrong:

- `src/env/e0.py:90` — `observe() -> np.ndarray` of shape `(grid_size,)`
  (a **pressure vector** over the grid, not four scalars).
- `src/experiments/__init__.py:37` — `RunResult` exposes only aggregates
  (`redistribution_mean/std/sum`), not per-tick trajectories.
- The (U, F, R, P) decomposition is a `doc_base/` lexicon item that was never
  instrumented in v0.1.0 code.

Coding against a factually wrong pre-registration would silently introduce a
new constraint-field component (forbidden by ADR-022 §8). Therefore this
correction. ADR-023's own §8 freeze clause requires a `.bis` for any change;
this is that document.

## 2. Scope of the correction

Only ADR-023 §4.1 is replaced. All other sections of ADR-023 — phase-space
metric (§4.3), library binding (§4.4), seed range `[1200, 1229]` (§4.5),
decision rule (§4.6), ADT-H6β tier (§5), implementation plan (§6),
anti-patterns (§7), freeze clause (§8) — remain in force.

ADR-022 is untouched. Its freeze SHA-256 (`86b49ac1…69cd068`) is intact.

## 3. Replacement of ADR-023 §4.1

**ADR-023 §4.1 (REPLACED).** Phase-space construction.

For each seed $s$:

1. Run Agent A on $E_0$ with `E0Config(seed=s)`. The horizon is
   $T = $ `E0Config.horizon` and the spatial dimension is
   $D = $ `E0Config.grid_size` (no value is hardcoded by ADR-023.bis;
   the H₅ defaults are inherited as-is).
2. At each tick $t \in [0, T)$, record:
   - the observed pressure vector $\mathbf{p}_t \in \mathbb{R}^D$ from
     `env.observe()` **before** the agent acts,
   - the integer action $a_t \in [0, D)$ emitted by `agent.select_action`.
3. Stack the pressure vectors into the trajectory matrix
   $\mathbf{X}_\Omega \in \mathbb{R}^{T \times D}$.
4. **Per-component z-score normalization** using per-run statistics
   $(\mu_d, \sigma_d)$ of column $d$ of $\mathbf{X}_\Omega$:
   $$\tilde{x}_{t,d} = (x_{t,d} - \mu_d) / \max(\sigma_d, 10^{-9})$$
   Justification: removes the spatial heterogeneity of the reference field
   (some grid cells carry baseline pressure orders of magnitude above others),
   while preserving topology by Φ₂-equivariance (validated by ADT-H6β-03).
5. The action sequence $\{a_t\}_{t=0}^{T-1}$ feeds the Markov null fit
   per ADR-022 §4.3 (Laplace $\alpha = 1$, alphabet size $D$).

Only `observe()` and `select_action` are read; no new sensor, no new field
component. The instrument is **strictly the H₅ public API**.

## 4. Resulting downstream impact

- ADR-023 §4.2 still applies: the null trajectory is built by replaying the
  Markov-sampled action sequence on a fresh $E_0$ with the same seed and
  recording pressure vectors the same way.
- ADR-023 §4.3 still applies: ripser on the (T, D) point cloud, persim
  bottleneck, mean over dim 0 and dim 1.
- The composite δ definition is unchanged.
- `D` is now whatever `E0Config.grid_size` is at run time (no value bound
  by ADR-023.bis; H₅ default propagates).

## 5. ADT impact

The five ADT-H6β tests (ADR-023 §5) remain valid as written. The synthetic
phase-space data in those tests was already 4-D-or-larger by design; no test
needs amendment. ADT-H6β-03 (z-score invariance) becomes the **load-bearing
guard** that the per-column normalization in §3.4 above does not destroy
the metric's scale-equivariance.
