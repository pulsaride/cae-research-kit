# ADR-022 — Pre-registration of H₆ (Adaptive Field Theory pivot)

**Status:** PROPOSED — not yet implemented
**Date:** 2026-05-09
**Supersedes scope of:** ADR-021 (H₅ falsification, archived)
**Confidentiality:** PRIVATE — `.forge_private/` only. Public release of this

## 1. Context

H₅ was rejected on 2026-05-09 (verdict `H5_REJECTED`, fingerprint
`406ce26ec3aeefada7e2250f16d24a89361c1da2041c6775599be394008e7e5f`):

- A vs B: Cohen's d = 0.145, Wilcoxon p = 0.237 → **indistinguishable**.
- A vs R: d = 0.595, p = 5.55e-04 → significant but underpowered.
- A vs P: d = 6.293, p = 1.86e-09 → trivial (P degraded).

The Wasserstein-1 redistribution metric on the cumulative pressure field cannot
distinguish online adaptation (A) from a well-tuned scripted policy (B).

The doctrine (`doc_base/idea-exchange.md`) anticipated this exact outcome and
prescribes a pivot: stop comparing agents, start measuring environment
deformation under any optimization pressure.

## 2. The pivot — what changes

| Dimension | H₅ (rejected) | H₆ (proposed) |
|---|---|---|
| Object of measurement | The agent | The environment E₀ |
| Comparison axis | A vs {B, R, P} | E₀ baseline vs E₀ post-exposure |
| Metric | Wasserstein-1 on pressure field | Persistent homology + entropy anisotropy |
| Null model | "Random walk" | "Stationary Markov process on E₀" |
| Falsification target | "A is special among agents" | "Optimization pressure deforms E₀ topologically" |

This is a **weaker claim**, intentionally. The H₅ formulation tried to identify
the optimizer by its signature; H₆ only asks whether optimization pressure of
any kind leaves a topological scar in the environment that a stationary Markov
process cannot reproduce.

## 3. H₆ — formal statement

> **H₆.** For a controlled environment E₀ exposed to a stream of actions from
> any non-stationary optimizer Ω, the persistent homology of the constraint
> field (U, F, R, P) at exposure end is **not** generated by a stationary
> first-order Markov process fitted to the same action distribution, with
> effect size $d \geq 0.5$ on the bottleneck distance and $p < 0.01$ over
> $n \geq 30$ disjoint seeds.

**Null hypothesis $H_0$:** the bottleneck distance between persistence diagrams
of (E₀ post-Ω) and (E₀ post-MarkovFit(Ω)) is statistically indistinguishable
from the bootstrap distribution under the Markov fit.

**Decision rule:**
- $d \geq 0.5$ AND Wilcoxon p < 0.01 over n=30 ⇒ `H6_SIGNATURE_DETECTED`
- $d < 0.2$ ⇒ `H6_REJECTED`
- otherwise ⇒ `H6_INCONCLUSIVE`

## 4. Pre-registered protocol (frozen before any code is written)

### 4.1 Environment
- E₀ identical to v0.1.0 (8–15 nodes, FastAPI + SQLite skeleton).
- No modification to `src/env/e0.py` allowed for H₆ v1.

### 4.2 Optimizer Ω
For H₆ v1, Ω = Agent A (adaptive) from v0.1.0. **Single optimizer.** Multi-Ω
generalisation is explicitly out of scope (would need ADR-023).

### 4.3 Markov baseline (the null)
For each seed:
1. Run A on E₀ for $T$ ticks → action sequence $\{a_t\}$.
2. Fit a first-order Markov chain $\hat{M}$ on $\{a_t\}$ (transition matrix
   estimated by counts + Laplace smoothing $\alpha=1$).
3. Sample $T$ actions from $\hat{M}$ → null sequence $\{a^0_t\}$.
4. Replay $\{a^0_t\}$ on a fresh E₀ instance with the same seed → null
   constraint field.

### 4.4 Topological metric
- Filter constraint field by sublevel set on each component (U, F, R, P
   independently; 4 diagrams per run).
- Compute persistence diagrams using `gudhi` (pinned `==3.9.0`) or `ripser`
   (pinned `==0.6.10`). **Library decision must be frozen before first run.**
- Bottleneck distance per component → average across {U, F, R, P}.
- Composite metric: $\delta = \overline{d_B(\text{PD}_\Omega, \text{PD}_{M_0})}$.

### 4.5 Sample size, seeds
- $n = 30$ disjoint seeds, range `[1100, 1129]`.
- Disjoint from H₅ seeds `[1000, 1029]` and H₅ offline seed `99`.
- Disjoint-seed guard mandatory in `src/experiments/h6/__init__.py`.

### 4.6 Determinism
- `CAE_SEED=42` for fingerprint, per-run seeds drawn from the disjoint range.
- New reference fingerprint required for H₆ image; do **not** reuse v0.1.0
  fingerprint.
- `PYTHONHASHSEED=42`, BLAS threads = 1, image SHA-256 pinned.

## 5. Adversarial Destruction Tests (mandatory, frozen pre-execution)

The H₆ ADT must include — and it must be written **before** the H₆ harness:

| ADT | What it tries to break | Pass condition |
|---|---|---|
| ADT-H6-01 | Bottleneck distance is non-zero on **same** sequence replayed twice | $\delta < 0.05$ |
| ADT-H6-02 | Markov-fit distance is non-zero on a sequence drawn FROM the Markov chain itself | $\delta < 0.1$ |
| ADT-H6-03 | Permutation test: shuffling action timestamps destroys $\delta$ | post-shuffle $\delta < 0.2$ |
| ADT-H6-04 | $\delta$ is invariant under endpoint relabeling (Φ₁) | $\| \delta_\Phi - \delta \| < 0.05$ |
| ADT-H6-05 | $\delta$ is invariant under temporal rescaling (Φ₂) | same |
| ADT-H6-06 | Replay with deterministic policy (B) produces $\delta < 0.2$ (sanity: scripted = no scar) | required |
| ADT-H6-07 | Random walk (R) produces $\delta < 0.2$ | required |

**If any ADT fails, the H₆ verdict is `H6_INCONCLUSIVE` regardless of the main
test outcome.** This is non-negotiable.

## 6. What would constitute scientific success

H₆ "survives" if and only if:
1. Main test: `H6_SIGNATURE_DETECTED` on n=30.
2. ADT-H6-01 through 07 all pass.
3. Replication on a second optimizer Ω' (e.g. a small RL agent, planned in
   ADR-023) shows $\delta_{\Omega'}$ in the same regime ($d \geq 0.5$).

Anything weaker is `H6_INCONCLUSIVE`. **Do not weaken the criteria post-hoc.**

## 7. What would constitute scientific failure

- $d < 0.2$ or any ADT fails ⇒ `H6_REJECTED`.
- Publish the null result with the same protocol as H₅ (RELEASE.md, verdict
  JSON, frozen image, ADR-024 documenting the falsification).
- The hypothesis space then becomes: **no environment-side topological
  invariant of optimization pressure exists at the scale of E₀**. This is a
  meaningful negative result and closes the kit at v0.2.0-final.

## 8. Anti-patterns explicitly forbidden

- Adding components to the constraint field beyond {U, F, R, P}.
- Replacing bottleneck distance with a more permissive metric mid-experiment.
- Increasing n beyond 30 to chase significance ("seed shopping").
- Reframing "Markov-fit baseline" as anything other than a stationary
  first-order chain on the same alphabet.
- Anthropomorphic vocabulary in commits, ADRs, or output JSON. ADR-020 stays
  active.
